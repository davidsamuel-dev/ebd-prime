--------------------------------------------------------------------------------
0.1 Objectivo do produto
--------------------------------------------------------------------------------
Aplicação móvel (React Native / Expo) para gestão da Escola Bíblica Dominical
(EBD): cadastro de congregação, turmas, alunos e professores; escala de aulas;
sessão de aula com frequência, materiais (Bíblia/revista), visitantes, oferta e
relatório; resumos por turma; histórico e aniversariantes; área "Geral" com
painel após primeira aula concluída; autenticação via API PHP+MySQL OU
Firebase (Firestore + Authentication), incluindo onboarding de nova escola e
recuperação de senha (REST ou e-mail Firebase).


--------------------------------------------------------------------------------
0.2 Stakeholders e perfis de utilizador
--------------------------------------------------------------------------------
[IMPL] Administrador da escola (onboarding Firebase ou fluxo demo).
[IMPL] Utilizador autenticado com `nivel_acesso` (ex.: admin, professor, …) e
       `congregacao_id` no contexto da sessão (`mobile/context/AuthContext.tsx`).
[IMPL] Modo demonstração sem backend (`authMode: demo`).
[FUTURO] Secretário dedicado, níveis hierárquicos "campo" vs "filial" com regras
         de visibilidade completas (ver RN legado na secção 6).


--------------------------------------------------------------------------------
0.3 Arquitectura actual (implementada)
--------------------------------------------------------------------------------
[IMPL] Cliente único em `mobile/` (Expo Router, TypeScript).
[IMPL] Camada de dados abstracta em `mobile/lib/api.ts`:
       - Se `getActiveDataBackend() === 'firebase'` → delega em
         `mobile/lib/firebase-backend.ts` (Firestore + Auth SDK).
       - Senão → HTTP JSON para `EXPO_PUBLIC_API_URL` (PHP em `backend/api/`).
[IMPL] Backend PHP 8 + PDO + MySQL (`backend/`), OpenAPI em
       `backend/openapi.yaml` (referência).
[IMPL] Regras Firestore em `firestore.rules` (raiz do repositório).
[PARCIAL] Pasta `/web` espelhada do mobile — NÃO existe neste repositório; só
          mobile + backend + docs.

1.1 Autenticação, sessão e recuperação de senha
--------------------------------------------------------------------------------
RF001  [IMPL] Ecrã de login (`mobile/app/login.tsx`): utilizador e senha; estado

relatórios, frequência, histórico, aniversariantes, resumo turma (ver `api.ts`).
